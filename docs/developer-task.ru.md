# Прямое задание для ИИ-разработчика

Ты работаешь над Sheepfold Family Internet Control.

Твоя задача — превратить текущий каркас в рабочий self-hosted продукт для OpenWRT-роутера с LuCI-интерфейсом, Android-приложением для родителей-администраторов и опциональным управлением через Telegram или MAX.

Перед началом работы прочитай:

- `AGENTS.md`;
- `docs/product-requirements.md`;
- `docs/security.md`;
- `docs/privacy.ru.md`;
- `docs/messaging.ru.md`;
- `docs/schedules.ru.md`;
- `docs/integrations.md`;
- `docs/domain-allowlist.ru.md`;
- `docs/ai-context-sharing.ru.md`.

## Главный сценарий

Родитель устанавливает Sheepfold на OpenWRT-роутер, открывает LuCI, видит все домашние устройства, настраивает белый список, чёрный список, расписания, временный доступ и аварийно-полезные сайты.

Android-приложение `Овчарня` дублирует основные родительские действия. Оно устанавливается только на телефоны родителей-администраторов.

Вне дома родитель может управлять короткими командами через один выбранный мессенджер: Telegram или MAX.

## Что сделать в первую очередь

1. Реализовать backend/API на роутере.
2. Реализовать LuCI-интерфейс с реальными вкладками:
   - белый список;
   - чёрный список;
   - все устройства;
   - расписания;
   - аварийно-полезные сайты;
   - Wi-Fi 2.4/5 GHz;
   - интеграции;
   - настройки;
   - журнал.
3. Реализовать обнаружение устройств из OpenWRT: DHCP leases, static leases, ARP/neighbor data.
4. Синхронизировать имена устройств и постоянные IP с OpenWRT DHCP static leases.
5. Реализовать правила firewall через `firewall4` / `nftables`.
6. Реализовать импорт/экспорт настроек, включая клиентов.
7. Реализовать безопасное обновление и удаление.
8. Реализовать Telegram-адаптер через outbound HTTPS long polling.
9. Оставить MAX-адаптер экспериментальным до проверки стабильной публичной Bot API.
10. После стабильного роутерного ядра проектировать Android-приложение.

## Жёсткие правила

- Устройство не может одновременно быть в белом и чёрном списке.
- Чёрный список всегда сильнее любых временных разрешений.
- Белый список не блокируется расписаниями и глобальной блокировкой.
- Устройства из чёрного списка не должны открывать LuCI, SSH и Sheepfold API.
- Заблокированные не по чёрному списку устройства могут видеть LuCI/API только если включена соответствующая настройка.
- Аварийно-полезные сайты выключены по умолчанию и включаются родителем.
- Не добавлять `yandex.ru`, маркетплейсы, супераппы, доставку еды, видео, игры, соцсети и app stores в дефолтный список.
- Яндекс Go / Яндекс Такси не предлагать по умолчанию.
- Android-приложение не ставится на телефоны детей.
- Не вводить облако разработчика.
- Не хранить пароль root роутера в Android.
- Не отправлять MAC/IP/имена детей/устройств/семейный контекст/логи ИИ-провайдеру без отдельного preview и подтверждения.

## OpenWRT

Цель — современные OpenWRT с `firewall4` / `nftables`.

Не реализовывать `firewall3` / `iptables`, если владелец проекта явно не попросит позже.

Установщик должен:

- требовать согласия с пользовательским соглашением;
- определить OpenWRT;
- определить наличие AdGuard Home и Podkop;
- выбрать рекомендуемый `integration_mode`;
- не перезаписывать настройки AdGuard Home/Podkop без отдельного подтверждения.

## LuCI

LuCI — основной интерфейс настройки.

Он должен быть понятным для родителя, а не только для сетевого администратора.

Все опасные действия требуют подтверждения:

- перезагрузка роутера;
- обновление;
- импорт настроек;
- очистка журнала;
- массовая блокировка;
- смена Wi-Fi имени/пароля;
- смена активного мессенджера;
- добавление администратора.

## Android

Минимум: Android 9.0 / API 28.

Первый запуск:

1. Показать пользовательское соглашение и чекбокс согласия.
2. Настроить подключение к роутеру.
3. Настроить локальную защиту приложения.
4. Рекомендовать пароль/PIN, а не биометрию.

Полный интерфейс работает локально с роутером. Удалённое управление без VPN/облака — через Telegram/MAX-бота.

## Бот

На роутере одновременно активен только один мессенджер: Telegram или MAX.

Telegram:

- реализовать через `getUpdates` long polling;
- не открывать входящие порты;
- хранить offset локально;
- принимать команды только от разрешённых администраторов.

MAX:

- держать как экспериментальный адаптер до подтверждения актуальной публичной Bot API.

## Логи

Дефолт:

```text
retention: 3d
max size: 1024 KB
masking: balanced
```

Не писать секреты, пароли, токены, session cookies, полные переписки, полные AI-запросы, историю сайтов, DNS-query-log всех устройств, банковские и медицинские данные.

## Интеграции

Целевая цепочка:

```text
Sheepfold -> AdGuard Home -> Podkop
```

Поддерживаемые режимы:

```text
none
adguard
podkop
adguard_podkop
```

AdGuard Home и Podkop не взаимоисключают друг друга.

## Definition Of Done

Работа считается готовой только если:

- изменения соответствуют `AGENTS.md`;
- обновлены EN/RU документы, если меняется пользовательское поведение;
- опасные действия требуют подтверждения;
- нет новых дефолтов, которые открывают развлечения или маркетплейсы;
- shell-скрипты проходят `bash -n`;
- JSON-файлы валидны;
- изменения не ломают существующий пользовательский конфиг.
